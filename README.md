# 🐺 狼人杀 AI 联机版

基于原版AI狼人杀系统，添加远程联机功能的多人在线版本。

## 功能特性

### 原版功能（全部保留）
- 4种游戏模式：经典8人局、猎人局、10人速推局、12人标准场
- API全自动模式：AI玩家通过API自动游戏
- 管理员模式：自定义发牌、查看所有身份、击杀/复活
- 性格系统：11种预设性格 + 自定义性格
- AI字数限制、AI强化模式
- 发言顺序设置（随机/1号开始/标准）
- 遗言系统、死后身份公开设置
- 游戏日志、投票详情记录
- 加密/明文双模式提示词

### 新增联机功能
- **房间系统**：创建公开房间或含6位密码的私人房间
- **房间列表**：浏览并快速加入公开房间
- **实时同步**：基于Socket.IO的实时游戏状态同步
- **房主控制**：房主负责游戏设置和流程控制（默认1号位）
- **📢 加速喇叭**：所有玩家可点击，给正在运行的AI发送竞速副本
  - 点击后为每个正在等待API回复的AI发送一个重复请求
  - 原请求和加速副本竞速，采纳最先完成的结果
  - 立即中断同一AI的其他未完成请求
  - 可多次点击发送多个副本

## 部署方式（Railway + GitHub）

### 1. 上传到 GitHub
```bash
# 创建新仓库，将解压后的所有文件推送到仓库
git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/YOUR_USERNAME/werewolf-online.git
git push -u origin main
```

### 2. 部署到 Railway
1. 访问 [railway.app](https://railway.app)
2. 点击 "New Project" → "Deploy from GitHub Repo"
3. 选择你的 werewolf-online 仓库
4. Railway 会自动检测 Node.js 项目并部署
5. 部署完成后会生成一个链接，分享给好友即可联机

### 环境变量（可选）
- `PORT`：服务器端口（Railway自动设置）

## 使用说明

### 房主（创建房间的人）
1. 输入名字 → 创建房间（公开/私人）
2. 将房间号分享给好友
3. 设置游戏模式、选项、API配置等（和原版一样）
4. 所有人到齐后点击"开始游戏"
5. 房主操控完整的游戏流程

### 其他玩家
1. 输入名字 → 输入房间号加入（私人房间需要密码）
2. 等待房主开始游戏
3. 在自己的界面上看到游戏进展和自己的角色信息
4. 在轮到自己时进行发言、投票、夜晚行动等操作

### 📢 加速喇叭
- 游戏过程中右下角有一个黄色喇叭按钮
- 当AI玩家正在等待API回复时，点击喇叭会为所有正在运行的AI发送加速副本
- 原请求和加速请求竞速，最先返回的结果被采用
- 适合在API响应缓慢时使用

## 技术架构

```
├── server.js          # Express + Socket.IO 服务器
├── package.json       # Node.js 依赖
├── public/
│   └── index.html     # 完整的游戏客户端（单文件）
└── README.md
```

- **后端**：Node.js + Express + Socket.IO
- **前端**：React (CDN) + Tailwind CSS + Socket.IO Client
- **API代理**：服务器端代理API调用，支持加速竞速和中断
- **房间管理**：服务器维护房间状态，支持断线重连

## 注意事项

- API密钥存储在浏览器localStorage中，不会传输给游戏服务器（仅通过代理转发）
- 房主的浏览器承担主要的游戏逻辑计算
- 建议使用稳定的网络环境
- 支持手机和电脑浏览器
